# Phase 12 — UI Design Contract

> Visual and interaction contract for the Unified Chat (Intent Classifier & Unified Route). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source of truth:** This phase REUSES the established light-theme design system already in production (v1.0–v1.2). Tokens below are DERIVED from `apps/web/src/styles/globals.css` — they are not new inventions. Do NOT introduce a new palette, typography, or component family. New surfaces (intent pill, override dropdown, header context selector, render dispatcher, table stub) must compose existing tokens and class patterns.

## Spacing Scale

Declared values (all multiples of 4; these are the values actually used across `globals.css`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pill internal gaps (`tool-pill` gap 5–6px → snap to 4), tight rows |
| sm | 8px | Compact element spacing (`chat-exchange` gap, footer gaps, chip gaps) |
| md | 12px | Card/panel internal padding rows, dropdown padding (`account-menu` padding 8px, menu-item 8–10px) |
| lg | 16px | Default element spacing — `assistant-card` / `tool-panel` / `output-box` padding, `tool-chat`/`chat-thread`-adjacent gaps |
| xl | 20px | Thread gap between exchanges (`chat-thread` gap: 20px) |
| 2xl | 24px | Topbar horizontal padding, section padding |
| 3xl | 32px+ | Page-level vertical rhythm (`workspace-content` uses `clamp(40px, 22vh, 240px)`) |

Exceptions:
- Border radii follow the existing system, NOT the spacing scale: pills/chips use `999px` (full round), chat input box `16px`, cards `10px`, buttons/menus `6px`, inner buttons `8px`. The intent pill MUST use `border-radius: 999px` to match `.tool-pill`/`.metadata-row span`.
- Touch targets: send button is 32×32px; ghost/copy buttons `min-height: 32–36px`; primary button `min-height: 40px`. New interactive controls (pill, dropdown trigger, header selector) MUST be ≥28px tall to match `.chat-mode-tab` / `.chat-lang-btn` minimums.

## Typography

Derived exactly from `globals.css`. The app is intentionally small-type (base `14px`). Declared roles (4):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Code / monospace | 13px | 400 | 1.5 (pre, `white-space: pre-wrap`) |
| Label / meta | 12px | 650 (semibold) | 1.5 — pills, metadata badges, field labels, menu labels |
| Heading (panel/section) | 15px | 700 | 1.25 — `output-header h2`, `topbar-brand` |

Weights (2): **400 regular** + **650 semibold** (the codebase uses `650`, not `600`; honor it). Display/auth headings use `20px/700` but are out of scope for this phase.

**Phase-specific application:**
- Intent pill label: 12px / 650 (matches `.tool-pill[data-active]` and `.metadata-row span`).
- Override dropdown items: 14px / 400 body (matches `.menu-item`).
- Header context selector labels/values: 12px (matches `.chat-compact-select`).
- Table-stub bridge message: 14px / 400 body inside an `assistant-card`.

## Color

The established palette (from `:root` in `globals.css`). The app is surface-heavy (white cards on near-white background) with a single green accent.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `#f7f8fa` | App background, `quota-blocked` bg, inactive button bg, mode-tab track |
| Secondary (30%) | `--surface` `#ffffff` | Cards (`assistant-card`, `tool-panel`), topbar, dropdown/menu surfaces, chat input box |
| Accent (10%) | `--primary` `#0b6b57` (green) | See reserved-for list |
| Destructive | `--destructive` `#b42318` | "Apagar histórico" confirmation only |
| Info (semantic, not accent) | `--info` `#2f6fed` | Streaming border state, note-block left rule, links — pre-existing semantic; reuse as-is |
| Warning (semantic) | `--warning` `#b54708` | Quota warnings, truncation badge — pre-existing semantic; reuse as-is |
| Text / Muted | `--text` `#111827` / `--muted` `#5e6a75` | Primary text / secondary text, pill text, placeholders |
| Border | `--border` `#d9dee5` | All hairline borders, card/pill/dropdown outlines |

Accent (`--primary` green) reserved for:
- The send button active state (`.chat-send-btn[data-active]`) — primary CTA of the input.
- The **active/selected** intent pill state and the **selected** option in the override dropdown (use the existing `rgb(11 107 87 / 8%)` tint + `--primary` text, exactly like `.tool-pill[data-active="true"]`).
- Active state of the header context selector segments (mirror `.chat-mode-tab[aria-selected]` tint).
- Copy button text/hover, Pro badge, focus outlines (`outline: 2px solid var(--primary)`), `topbar-brand`.
- NOT for: the resting intent pill (resting = `--surface` bg, `--muted` text, `--border` outline), output-panel chrome, or body links (those use `--info`).

**Output distinguishability (UNI-03) — color is NOT the differentiator.** Heterogeneous outputs in one thread are distinguished by *structure + a labeled header*, not by inventing per-type colors:
- Code-bearing outputs (formula / sql / regex / script) → reuse `.output-box` with monospace `<pre>` and the existing `data-status` border tint (`--info` while streaming, `--primary` on complete).
- Explanatory text (analysis) → reuse `.output-box` prose (`<p>`/`<ol>`), no `<pre>`.
- Table stub (`table_stub`) → reuse `.placeholder-box` (dashed `--border`, `#fbfcfd` bg, `--muted` text) to read clearly as "coming next", distinct from a real answer card.
- Each output card carries a small type label in the `.output-header` / `.metadata-row` (the intent pill itself), so the type is legible without relying on hue.

## Interaction Contracts (phase-specific surfaces)

> Beyond tokens, these define the behavior the executor must implement. Forms exact-but-derived from existing components.

### Intent Pill (UNI-02)
- **Placement:** Above the assistant output card, left-aligned, as the first element of each assistant `.chat-exchange` (visible the moment the `intent_detected` NDJSON event arrives — before streaming text, per RESEARCH Pitfall 7).
- **Resting style:** `.tool-pill` look — `999px` radius, `--surface` bg, `--border` outline, `--muted` text, 12px/650, leading lucide icon at `size={13}`.
- **Active/overridden style:** `.tool-pill[data-active="true"]` look — `--primary` text, `rgb(11 107 87 / 8%)` bg, `--primary` border.
- **Affordance:** entire pill is a `<button aria-haspopup="listbox" aria-expanded>`; a trailing chevron (`ChevronDown size={12}`) signals it opens a menu.
- **During streaming:** pill shows detected type immediately; remains interactive (override allowed mid/after stream — re-run replaces the output).

### Override Dropdown (UNI-02 / D-03)
- **Pattern:** reuse the `.account-menu` popover pattern (absolute, `top: calc(100% + 8px)`, `--surface` bg, `--border`, `box-shadow: 0 8px 24px rgb(15 23 42 / 12%)`, `role` listbox/menu, Escape + outside-click to close — mirror Topbar's popover effect handling).
- **Items:** the 7 intent options, each a `.menu-item` (8–10px padding, lucide icon + label, hover `--background`). The currently active type shows the `--primary` selected tint.
- **On select:** closes the menu and **immediately re-submits** the original prompt with `overrideIntent` (D-03 — re-runs generation, not relabel). Show "Gerando..." status on the re-run.

### Header Context Selector (UNI-05 / D-01 / D-02)
- **Placement:** in the `Topbar` `.topbar-actions` row, left of "Nova conversa" — persistent across turns.
- **Form:** compact segmented controls / selects matching `.chat-mode-tabs` + `.chat-compact-select` styling (28px tall, 12px text). Plataforma + Idioma da fórmula as segmented toggles (`.chat-lang-toggle` pattern); Separador and Dialeto SQL as compact selects.
- **Behavior:** header value is an explicit override of AI-inferred context; persists in session state (not reset per message). Reflect resolved context in the existing `.metadata-row` of each output (platform / formulaLanguage / separator), exactly as `FormulaOutputPanel` already does.

### Render Dispatcher (UNI-03)
- Switch on `payload.kind`: `formula`/`explanation` → existing `FormulaOutputPanel` family; `sql` → `SqlOutputPanel`; `regex`/`script`/`template` → their existing output panels (all reuse `.output-box`); `table_stub` → `.placeholder-box` stub; `needs_file` event → `assistant-card` needs-file prompt with paperclip CTA.
- No new card chrome: every branch lands inside `.assistant-card` so the thread stays visually uniform; the intent pill + metadata row carry the type signal.

### Sidebar / Tool access (UNI-07)
- `ToolNav` (the `.tool-pill` row) remains the discovery surface — per-tool deep links stay accessible. The unified chat is the default `/workspace` entry; do NOT remove ToolNav in this phase (final navigation migration is Phase 15). The active-state styling of `.tool-pill[data-active]` is unchanged.
