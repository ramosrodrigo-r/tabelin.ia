<objective>
Implementar a rota `/api/chat/unified` que classifica o intent, emite o pill de intent via NDJSON, despacha para os resolvers existentes e persiste histórico no `toolKind` resolvido.

Purpose: Esta é a integração central da Phase 12. O usuário digita uma vez no chat unificado; o backend decide o resolver correto, preserva quota/pro-gate/attachment/history existentes e mantém o SLA por emitir `intent_detected` como primeiro evento da resposta.

Output: route handler unified, serialização `table_stub` em contexto multi-turn e testes de rota.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> `/api/chat/unified` | Prompt, file, context selectors, override, and lastIntent are user-controlled |
| Route -> resolver | Route normalizes request shape and selects resolver server-side |
| Route -> persistence | Assistant payload and attachmentContext are persisted per `userId+toolKind` |
| Extracted file -> LLM | Extracted content remains untrusted and must use existing delimiter injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-02-01 | Spoofing | Client-supplied tool kind | mitigate | Client never sends `toolKind`; route derives `resolvedToolKind` from validated intent map only. |
| T-12-02-02 | Tampering | `overrideIntent` | mitigate | Reuse `overrideIntentSchema`; forced override changes dispatch only after enum validation. |
| T-12-02-03 | Elevation of Privilege | File attachment in unified route | mitigate | Run `getUserEntitlement` pro-gate before `reserveToolUse` and before `extractContent`. |
| T-12-02-04 | Information Disclosure | Cross-tool history | mitigate | Load and save history under resolved `toolKind`; never save all exchanges under a generic `unified` key. |
| T-12-02-05 | Denial of Service | Oversized prompt/file | mitigate | Reject prompt >8000 chars before provider work; preserve existing 5 MB file cap and extraction failure release path. |
| T-12-02-06 | Repudiation | Quota accounting | mitigate | Confirm quota only after resolver/table-stub success; release quota on extraction failure, needs-file, and catch-block failures. |
</threat_model>
