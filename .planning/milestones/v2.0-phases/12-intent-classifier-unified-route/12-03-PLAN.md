<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> browser state | Prompt, pasted text, selected file, and override clicks are untrusted |
| Browser -> `/api/chat/unified` | Client sends context selectors and override intent but server remains source of truth |
| NDJSON stream -> React render | Payloads are external data and must be parsed before rendering |
| File attachment -> browser preview | File name/size are browser-provided metadata |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03-01 | Tampering | NDJSON stream parsing | mitigate | Parse each line with `unifiedStreamEventSchema`; corrupted lines enter error state. |
| T-12-03-02 | Information Disclosure | Attachment preview | mitigate | Preview shows only local file name/size and extracted text returned by authenticated route; no raw file is persisted client-side beyond pending state. |
| T-12-03-03 | Spoofing | Intent override UI | mitigate | UI can request override, but server validates enum and maps toolKind; UI labels never decide persistence. |
| T-12-03-04 | Cross-Site Scripting | Heterogeneous output rendering | mitigate | Render through existing panels and text nodes; `TableIntentStub` and needs-file state use textContent/React escaping; no `dangerouslySetInnerHTML`. |
| T-12-03-05 | Denial of Service | Repeated override while streaming | mitigate | Disable duplicate submit while pending or abort/replace the active stream before re-submit; test that only one active live exchange is shown. |
</threat_model>
