## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→UnifiedChatTool | Usuário edita spec no ConfirmationCard — dados não confiáveis enviados para o servidor via campo specOverride |
| UnifiedChatTool→hook→fetch | overrideGenerate e specOverride (spec editada serializada) no body do POST — campos dedicados, prompt intacto |
| RenderDispatcher→ClarificationCard | Payload vem do servidor (confiável após parse Zod) mas é renderizado no browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-SC | Tampering | npm installs | accept | Phase 13 não instala pacotes novos |
| T-13-12 | Tampering | spec editada pelo usuário no ConfirmationCard | mitigate | spec viaja no campo dedicado specOverride e é re-validada com tableSpecPayloadSchema.safeParse no servidor (route.ts Plan 03 Task 1) antes de usar; fallback é spec reconstruída do histórico PostgreSQL |
| T-13-13 | Tampering | XSS em payload.question renderizado | mitigate | React renderiza payload.question em {payload.question} dentro de <p> — textContent automático; sem dangerouslySetInnerHTML; alinhado com SEC-05 |
| T-13-14 | Tampering | XSS em colunas do ConfirmationCard | mitigate | Colunas renderizadas via input value controlado (React) — sem innerHTML; textoContent automático |
| T-13-15 | Abuse | loop infinito via resubmit de clarificação | mitigate | Teto de turns (MAX_CLAR_TURNS=2) é server-side derivado do PostgreSQL; resubmits de clarificação incrementam o contador que será detectado no próximo request |
| T-13-16 | Elevation | overrideGenerate injetado pelo usuário | mitigate | Mesmo que o usuário manipule o campo no DevTools, o servidor re-verifica o path de geração com confirmToolUse — a cota é debitada corretamente no generation path independente da origem do override |
