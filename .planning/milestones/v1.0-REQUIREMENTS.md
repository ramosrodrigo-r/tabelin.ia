# Requirements: Tabelin.IA — v1.0 MVP

**Defined:** 2026-05-23
**Core Value:** Brazilian spreadsheet users can describe the outcome they need in Portuguese and quickly receive correct, copy-ready formulas, code, queries, or structured table outputs that fit their actual tools.

## v1 Requirements (ALL COMPLETE ✓)

### Authentication and Accounts

- [x] **AUTH-01**: User can sign up with email and password. — ✓ Phase 1
- [x] **AUTH-02**: User can sign in, sign out, and keep a session across browser refresh. — ✓ Phase 1
- [x] **AUTH-03**: User can reset a forgotten password. — ✓ Phase 1
- [x] **AUTH-04**: Authenticated user can access a workspace that preserves their plan and usage state. — ✓ Phase 1

### Quotas and Billing

- [x] **QUOT-01**: Free user is limited to 4 tool uses per 12-hour window. — ✓ Phase 2 (reserve/confirm/release pattern)
- [x] **QUOT-02**: Free user is limited to 10 AI chat messages per 30-day window. — ✓ Phase 2 (modeled, enforcement deferred to v2 rollout)
- [x] **QUOT-03**: Upload limits are enforced for free users: 5 MB per file and 5 files per history. — ✓ Phase 2 (modeled, UI enforcement deferred to v2)
- [x] **BILL-01**: User can start Pro checkout with Pix and Brazilian card support. — ✓ Phase 2 (Mercado Pago Checkout Pro)
- [x] **BILL-02**: Payment webhook activates, updates, or revokes Pro entitlement idempotently. — ✓ Phase 2
- [x] **BILL-03**: Pro user has unlimited access to formula, script, SQL, regex, OCR, and file-analysis tools subject to abuse safeguards. — ✓ Phase 2

### Tool Workspace

- [x] **WORK-01**: User can navigate a distraction-free sidebar workspace with Formula, Scripts, SQL, Regex, File Analysis, and OCR tools. — ✓ Phase 5 (all 6 tools active)
- [x] **WORK-02**: Every tool uses a consistent input, streaming output, error, and loading pattern. — ✓ Phases 1–5
- [x] **WORK-03**: Every generated formula, code block, SQL query, regex, chart data, report, and table output has a prominent copy button with immediate copied feedback. — ✓ Phases 1–5
- [x] **WORK-04**: Tool outputs show assumptions or warnings when the request lacks enough context for safe use. — ✓ Phase 1, 3

### Formula Assistant

- [x] **FORM-01**: User can describe a spreadsheet task in Portuguese and receive a generated formula. — ✓ Phase 1
- [x] **FORM-02**: User can choose the target platform: Microsoft Excel, Google Sheets, Airtable, or LibreOffice Calc. — ✓ Phase 1
- [x] **FORM-03**: User can choose formula language explicitly: Portuguese (Brazil) with `;` separators or English with `,` separators. — ✓ Phase 1
- [x] **FORM-04**: User can paste an existing formula and receive a step-by-step Portuguese explanation. — ✓ Phase 1
- [x] **FORM-05**: Formula output includes platform, formula language, separator, and assumptions so the user can verify paste compatibility. — ✓ Phase 1

### Code, SQL, and Regex Tools

- [x] **CODE-01**: User can generate VBA scripts for Excel automation from a Portuguese prompt. — ✓ Phase 3
- [x] **CODE-02**: User can generate Google Apps Script for Sheets automation from a Portuguese prompt. — ✓ Phase 3
- [x] **CODE-03**: User can generate Airtable Scripts from a Portuguese prompt. — ✓ Phase 3
- [x] **SQL-01**: User can generate SQL queries from text prompts. — ✓ Phase 3
- [x] **SQL-02**: User can select SQL dialect: PostgreSQL, MySQL, SQL Server, Oracle, or BigQuery. — ✓ Phase 3
- [x] **REGX-01**: User can generate regex patterns from Portuguese prompts. — ✓ Phase 3
- [x] **REGX-02**: User can paste an existing regex and receive a Portuguese explanation. — ✓ Phase 3
- [x] **SAFE-01**: Generated scripts and SQL include warnings or guardrails for destructive operations. — ✓ Phase 3

### Spreadsheet File Analysis

- [x] **FILE-01**: User can upload `.csv` and `.xlsx` files up to 5 MB. — ✓ Phase 4
- [x] **FILE-02**: System extracts sheet names, headers, inferred column types, and representative sample rows. — ✓ Phase 4
- [x] **FILE-03**: User can chat against the uploaded file's detected schema and sample data. — ✓ Phase 4
- [x] **FILE-04**: User can request text pivot-table style summaries from uploaded spreadsheet data. — ✓ Phase 4
- [x] **FILE-05**: User can request executive insight reports from uploaded spreadsheet data. — ✓ Phase 4

### Charts and OCR

- [x] **CHRT-01**: User can request chart suggestions for uploaded spreadsheet data. — ✓ Phase 5
- [x] **CHRT-02**: User can render bar, line, and pie charts in the frontend using parsed spreadsheet data. — ✓ Phase 5
- [x] **OCR-01**: User can upload `.png`, `.jpeg`, or `.jpg` images that contain tables. — ✓ Phase 5
- [x] **OCR-02**: System reconstructs image tables into structured rows and columns. — ✓ Phase 5 (OpenAI Vision gpt-4o-mini)
- [x] **OCR-03**: User can copy reconstructed tables as TSV or CSV for direct paste into Excel. — ✓ Phase 5

### Privacy, Performance, and Reliability

- [x] **PRIV-01**: Uploaded raw files are deleted when the chat ends or after 1 hour of inactivity, whichever comes first. — ✓ Phase 4 (node-cron cleanup)
- [x] **PRIV-02**: Raw uploaded file contents are not written to application logs. — ✓ Phase 4
- [x] **PRIV-03**: The product documents that customer data is not used for public model retraining by selected commercial AI providers unless explicitly opted in. — ✓ Phase 4 (privacy page)
- [x] **PRIV-04**: Provider file uploads use available expiration controls when raw files must be sent to a provider. — ✓ Phase 4
- [x] **PERF-01**: Simple formula generation begins streaming visible output within 2.5 seconds under normal provider latency. — ✓ Phase 1
- [x] **RELY-01**: AI outputs are validated against structured schemas before the UI presents copy-ready content. — ✓ Phase 1 (Zod validation throughout)

### Pro Value and Support

- [x] **PRO-01**: Pro user can access advanced table template generation. — ✓ Phase 3
- [x] **PRO-02**: Pro user can see support contact paths for priority email or WhatsApp support. — ✓ Phase 2
- [x] **PRO-03**: Pro requests can be marked for priority processing when infrastructure supports prioritization. — ✓ Phase 2
