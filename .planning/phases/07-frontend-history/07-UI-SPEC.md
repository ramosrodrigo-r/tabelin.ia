# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase reuses the entire established design system from v1.0. Net-new UI is limited to:
1. A **"Nova conversa"** button in the workspace topbar (`components/app/topbar.tsx`), consistent across all 5 text tools (D-06).
2. A **confirmation popover** anchored beneath that button, reusing the existing `.account-menu` dropdown visual pattern (D-05, resolved).

Everything else — chat-thread rendering of restored exchanges, restored selectors, empty state — uses existing components and tokens unchanged. Restored exchanges render through the existing `.chat-thread` / `.chat-exchange` / `.user-bubble` + `*-output-panel` patterns with no visual changes.

## Interaction Spec — "Nova conversa" Confirmation Popover (D-05)

Net-new interaction. Built entirely on the existing `.account-menu` dropdown pattern. Zero new dependencies.

**Trigger & anchoring**
- Triggered by the "Nova conversa" ghost button in the topbar.
- Popover opens **anchored beneath the trigger** using the existing pattern: `position: absolute; top: calc(100% + 8px);` within a `position: relative` container (mirrors `.account-menu-container` / `.account-menu`).
- NOT a centered modal. NOT `window.confirm()`.

**Visual (reuse `.account-menu` tokens)**
- Background `--surface` (#ffffff), border `1px solid --border` (#d9dee5), `border-radius: 6px`, `box-shadow: 0 8px 24px rgb(15 23 42 / 12%)`, `z-index: 10`, internal padding 8px.
- Prompt text: Body 14px / weight 400 / line-height 1.5, color `--text`.
- Action row: two buttons.
  - "Apagar histórico" — destructive style using `--destructive` (#b42318).
  - "Cancelar" — ghost style (`.ghost-button`).

**Accessibility**
- Container `role="dialog"` with an accessible label (`aria-label` or `aria-labelledby` referencing the prompt text).
- Focus is **trapped** within the popover while open.
- **Esc** closes the popover (cancel, no delete).
- On close (Esc, Cancelar, click-outside, or completed delete), focus **returns to the trigger** button.
- Trigger button exposes `aria-haspopup="dialog"` and `aria-expanded` reflecting open state.

**Behavior**
- "Cancelar" or Esc or click-outside: close popover, no state change.
- "Apagar histórico": call `DELETE /api/conversations/[tool]`, clear local exchanges state, close popover, return focus to trigger (hard delete per D-04).
