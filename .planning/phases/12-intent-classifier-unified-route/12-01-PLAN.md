<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser request body -> classifier | `prompt`, `lastIntent`, and `overrideIntent` are untrusted user-controlled data |
| Classifier -> route dispatcher | `intent` decides which resolver executes and which `toolKind` history is loaded |
| OpenAI response -> server code | Structured output is external provider output and must be schema-validated before use |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01-01 | Spoofing | `overrideIntent` | mitigate | Validate override against `overrideIntentSchema`; reject `unknown` and any string outside the enum before dispatch. |
| T-12-01-02 | Tampering | OpenAI classifier output | mitigate | Use `zodResponseFormat` parse path or explicit `intentClassificationSchema.parse` fallback; never trust raw JSON. |
| T-12-01-03 | Tampering | `lastIntent` / session context | mitigate | Delimit context with `[CONTEXTO ANTERIOR]` markers and instruct the classifier that prior context is reference data, not instructions. |
| T-12-01-04 | Denial of Service | Oversized prompt to classifier | mitigate | Route-level prompt length cap is implemented in 12-02; classifier also trims and handles invalid short prompts without provider calls. |
| T-12-01-SC | Supply Chain | Dependencies | accept | No new npm packages are added; `zodResponseFormat` already exists in installed `openai@6.39.0`. |
</threat_model>
