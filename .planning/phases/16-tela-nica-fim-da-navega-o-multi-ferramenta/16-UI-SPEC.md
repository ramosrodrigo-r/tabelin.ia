## Spacing Scale

Declared values (must be multiples of 4) — reaproveita os tokens já em uso no projeto:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gaps de ícone, espaçamento interno de pills/badges |
| sm | 8px | Espaçamento entre elementos compactos (toolbar do grid, gap entre painéis em mobile) |
| md | 16px | Padding padrão de painéis (sidebar/topbar antigos usavam 16px) |
| lg | 24px | Padding lateral do `workspace-content`/topbar, gap do split desktop |
| xl | 32px | Reservado — não usado diretamente nesta fase |
| 2xl | 48px | Reservado — não usado diretamente nesta fase |
| 3xl | 64px | Reservado — não usado diretamente nesta fase |

Exceptions:
- Toggle de alternância grade↔chat em telas estreitas (D-03): botões com altura mínima de 44px (alvo de toque), seguindo o padrão `.primary-button`/`.ghost-button` (`min-height: 40px`) ajustado para 44px neste componente específico.
- Topbar mantém `min-height: 56px` (já existente, não é múltiplo de 4 mas é valor herdado — não alterar).

## Typography

Reaproveita a escala global já declarada em `globals.css` (body `font-size: 14px`, `line-height: 1.5`). Nenhuma fonte/tamanho novo é introduzido nesta fase — apenas reorganização de layout.

Apenas **2 pesos** são usados nesta fase: **400 (regular)** para texto de corpo, e **700 (strong)** para qualquer elemento que precise de ênfase (labels, marca/heading da topbar, item ativo do toggle mobile). Onde o CSS existente usa 600 ou 650, consolidar para **700**.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 700 (strong — consolidado; `.field label` e `.attachment-chip-name` passam de 650 para 700) | 1.4 |
| Heading (topbar brand) | 15px | 700 (strong — já usado em `.topbar-brand`) | 1.2 |
| Display (não usado nesta fase) | 20px | 700 (strong) | 1.2 |

Marca "Tabelin.IA" na Topbar (D-12): manter `.topbar-brand` (15px / 700 / cor `var(--primary)`, `letter-spacing: -0.3px`) — o mesmo estilo já usado, apenas reposicionado/reaproveitado de `.sidebar-brand strong` se uma tagline for adicionada (12px / `var(--muted)`, igual `.sidebar-brand span`).

## Color

Reaproveita 100% as custom properties já definidas em `:root` de `globals.css` — nenhuma cor nova é introduzida.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` = `#f7f8fa` | Fundo da página, área de split entre painéis |
| Secondary (30%) | `var(--surface)` = `#ffffff` | Painel da planilha (`TableGridPanel`), painel do chat, topbar |
| Accent (10%) | `var(--primary)` = `#0b6b57` | Reservado para: marca "Tabelin.IA" na topbar, botão de envio ativo do chat (`--chat-btn-active-bg`), borda/seleção ativa do grid (`--dsg-selection-border-color`), estado ativo do toggle grade/chat em mobile |
| Destructive | `var(--destructive)` = `#b42318` | Apenas ações destrutivas — ex.: botão "Apagar histórico" já existente no menu da Topbar (não alterado nesta fase) |

Accent reserved for: marca "Tabelin.IA" (topbar), botão de enviar mensagem ativo no chat, indicador do item ativo no toggle grade↔chat (mobile), borda de seleção de célula no grid. Não usar accent em texto de corpo, bordas decorativas ou superfícies grandes.

## Layout Contract (specific to this phase)

### Desktop (≥ 901px — acima do breakpoint mobile existente de 600px; usar 900px como breakpoint do split, alinhado ao padrão de `@media (max-width: 600px)` já existente no projeto, estendido para cobrir tablets em retrato)

- `.workspace-body` (já existe) deixa de conter `Sidebar` + `main`. Passa a conter dois painéis lado a lado (D-01):
  - **Painel da planilha** (`.workspace-grid-panel`): `flex: 7` (≈70% — D-02), monta `<TableGridPanel>` persistente com planilha-amostra estática (D-04), `min-width: 0` para permitir scroll horizontal do grid sem estourar o layout.
  - **Painel do chat** (`.workspace-chat-panel`): `flex: 3` (≈30% — D-02), monta `<UnifiedChatTool>`, `border-left: 1px solid var(--border)`, `background: var(--surface)`.
  - Gap entre painéis: `0` (a borda `1px solid var(--border)` do painel do chat já define a separação visual — sem gap extra, conforme padrão da antiga `.sidebar` que usava `border-right` em vez de gap).
  - Cada painel usa `padding: 16px` (md) internamente; `overflow: auto` para scroll independente.
  - Sem divisória arrastável (D-02, deferred).

### Mobile / telas estreitas (< 900px)

- Toggle/abas para alternar entre "Planilha" e "Chat" (D-03) — mostra um painel por vez, ocupando 100% da largura.
  - Toggle fica fixo no topo do `.workspace-body`, abaixo da Topbar: dois botões "Planilha" / "Chat", `display: flex`, `border-bottom: 1px solid var(--border)`, `background: var(--surface)`.
  - Botão ativo: `color: var(--primary)`, `font-weight: 700` (consolidado — peso "strong" único da fase), `border-bottom: 2px solid var(--primary)` (reaproveita o padrão visual de `.chat-tab-active`/`.chat-tab-active-text` já definido em `:root`, ajustado para 700).
  - Botão inativo: `color: var(--muted)`, `font-weight: 400`.
  - Altura mínima de cada botão do toggle: 44px (alvo de toque — exceção à escala de 8pt).
  - Painel oculto via `display: none` (não desmontar componentes — preserva estado do grid/chat ao alternar).
  - Breakpoint sugerido: `@media (max-width: 900px)` — novo breakpoint específico desta fase (o existente de 600px permanece para ajustes de padding/topbar já feitos).

### Topbar enxuta (D-10, D-11, D-12)

- Mantém `.topbar` (56px, `border-bottom: 1px solid var(--border)`, `background: var(--surface)`, `padding: 0 24px`).
- Esquerda: marca "Tabelin.IA" (`.topbar-brand`, movida da Sidebar — D-12). Tagline opcional abaixo/ao lado em 12px `var(--muted)` se houver espaço (não obrigatório).
- Direita (`.topbar-actions`, mantido): sessão do usuário (email) + ação "Sair" (logout) + link para `/privacidade`. Billing/entitlement/Pro badge **permanecem intactos** (D-10 — saem na Phase 17, não nesta fase).
- Remover: `useWorkspaceToolKind()` e qualquer detecção de `toolKind` por rota (sem rotas de tool restantes para detectar).

### Sidebar e ToolNav

- `Sidebar` (`apps/web/src/components/app/sidebar.tsx`) é removida do layout (CLEAN-05). Classes `.sidebar`, `.sidebar-brand`, `.sidebar-nav` etc. podem ser removidas do CSS se não usadas em outro lugar (verificar antes).
- `ToolNav` já está órfão — pode ser removido nesta fase ou deixado para Phase 18 (decisão do planner).

### Rotas antigas (D-07)

- `/workspace/{sql,regex,scripts,templates,file-analysis,ocr}` → redirect 308 para `/workspace`. Sem UI nova — apenas comportamento de roteamento.
