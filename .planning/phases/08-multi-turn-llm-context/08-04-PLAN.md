<objective>
Corrigir o defeito de prompting que faz o modelo repetir a resposta anterior verbatim em
follow-ups multi-turn, em vez de aplicar a nova instrução. O bug tem duas causas
compounding, ambas na camada de prompting:

1. serializeAssistant injeta turnos anteriores como prosa sem rótulo, conflitando com o
   system prompt ("Responda APENAS com JSON") e o response_format json_object — o modelo
   não entende que aquela prosa é "contexto a modificar", não "o que devo reproduzir".

2. Os system prompts dos 4 tools são single-shot e nunca dizem ao modelo que mensagens
   anteriores são contexto de referência e que ele deve responder/refinar o ÚLTIMO pedido.

Purpose: Fechar os 2 gaps major do 08-UAT.md e fazer os testes 1 e 2 passarem.
Output:
  - serializeAssistant com rótulo "[Resposta anterior]" por tool
  - buildMultiTurnSystemPrompt helper DRY (context-messages.ts) que anexa parágrafo
    multi-turn ao system prompt quando há histórico
  - 4 stream files consumindo buildMultiTurnSystemPrompt
  - Novos testes de comportamento em multi-turn-context.test.ts com histórico não-vazio
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| assistantPayload → role:assistant content | Conteúdo gerado pelo LLM, persitido, reenviado como histórico — não confiável |
| userPrompt → role:user content (histórico) | Input do usuário salvo em turno anterior — não confiável |
| buildMultiTurnSystemPrompt | Concatena literal hardcoded com basePrompt; o basePrompt é literal do código, não input de usuário — confiável |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-GAP-01 | Tampering | serializeAssistant — rótulo "[Resposta anterior]" | accept | Rótulo é simples prefixo de texto; não altera o stripping de metadata/JSON cru já existente. Adversário que controla userPrompt/assistantPayload já tinha acesso ao prompt antes — o rótulo não amplia a superfície. |
| T-08-GAP-02 | Spoofing | buildMultiTurnSystemPrompt — instrução multi-turn hardcoded | accept | Texto é literal no código-fonte, não interpolado a partir de input externo. Não há vetor de injeção novo. |
| T-08-GAP-03 | Information Disclosure | Histórico rotulado revelando artefato anterior ao modelo | accept | O histórico já era enviado antes desta correção. O rótulo muda a forma (sinaliza "referência"), não amplia o que é enviado. Truncagem por tokens (T-08-02) e MAX_EXCHANGES preservados. |
| T-08-GAP-SC | Tampering | npm install (nenhum pacote novo neste plano) | accept | Nenhuma dependência nova adicionada — sem vetor de supply chain. |
