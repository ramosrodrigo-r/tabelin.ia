<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /api/tools/ocr/process | Campo imageBase64 é string não confiável do cliente — pode conter payload injetado ou imagem oversized |
| route.ts → ocr-processor.ts | Dados base64 passam para chamada OpenAI Vision — conteúdo da imagem pode conter texto de prompt injection |
| ocr-processor.ts → OpenAI API | Resposta JSON do AI é tratada como dado confiável após parse — precisa de validação antes de renderizar |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering | POST /api/tools/ocr/process | mitigate | ocrRequestSchema.safeParse valida mimeType como z.enum(["image/png","image/jpeg"]) — rejeita 400 para qualquer MIME não-listado; imageBase64 validado como z.string().min(1) |
| T-05-01-02 | DoS | POST /api/tools/ocr/process | mitigate | Validação server-side: imageBase64.length * 0.75 > 5MB → retorna 413 antes de chamar OpenAI; quota reserve impede abuso por usuário |
