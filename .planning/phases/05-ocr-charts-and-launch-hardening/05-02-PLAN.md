| Boundary | Description |
|----------|-------------|
| AI response → useFileChat complete event | Conteúdo do event.content é string não validada retornada pelo AI — pode ser JSON malformado ou injetado |
| useFileChat chartData → ChartMessage | chartData é passado como objeto para o componente React — deve ser validado antes de renderizar |
| Browser → pnpm add recharts | Instalação de pacote npm — verificar legitimidade antes de instalar |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | useFileChat complete event (JSON.parse de resposta AI) | mitigate | JSON.parse envolto em try/catch; verificação estrutural dos campos chartType, xKey, yKey, rows antes de criar msg chart; campos inesperados são descartados (mensagem vira type: "text") |
| T-05-02-02 | Tampering | ChartMessage — dados do AI renderizados como props React | mitigate | chartDataSchema Zod (criado no Plano 05-01) pode ser usado para safeParse antes de renderizar; dados são usados apenas como propriedades de componentes React (não executados como HTML ou eval) |
| T-05-02-03 | DoS | CHART_PROMPT → AI retorna JSON com rows gigante | accept | Recharts renderiza apenas os pontos passados; não há limite superior problemático para listas de até 1000 pontos em BarChart; datasets maiores são improvável dado limite de 5MB do arquivo de origem |
| T-05-02-04 | Spoofing | recharts npm package | mitigate | Package Legitimacy Audit verificado no RESEARCH.md — recharts [ASSUMED aprovado]: 10+ anos, MIT, github.com/recharts/recharts, npm view recharts versão 3.8.1; executor confirma npmjs.com/package/recharts antes de instalar; versão específica 3.8.1 via pnpm add recharts@3.8.1 se necessário |
| T-05-02-SC | Tampering | npm/pnpm install recharts | mitigate | Verificar legitimidade do pacote em npmjs.com antes da instalação; pacote marcado [ASSUMED] no RESEARCH.md — requer confirmação do executor antes de prosseguir |
