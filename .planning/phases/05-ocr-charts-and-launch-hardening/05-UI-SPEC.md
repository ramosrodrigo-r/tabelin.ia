## Spacing Scale

Declared values (multiples of 4) — carried forward from existing `globals.css` patterns:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, chip internal gap |
| sm | 8px | Element gap in rows, button internal gap, form field gap |
| md | 16px | Panel internal padding, section gap, drop zone padding |
| lg | 24px | Workspace content padding, section breaks |
| xl | 32px | Auth panel padding |
| 2xl | 48px | (reserved — not used in Phase 5 UI) |
| 3xl | 64px | (reserved — not used in Phase 5 UI) |

Exceptions:
- Touch target minimum: 40px height (`.primary-button` min-height, carried from Phase 1)
- Quick-action buttons: 32px effective height (4px padding + 12px font = ≈32px), consistent with existing `.quick-action-row` buttons in Phase 4

## Typography

Carried directly from `globals.css` body declarations. Consolidated to 4 sizes and 2 weights for Phase 5.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / Caption / Axis / Table cell / Chart title | 12px | 600 (semibold) | 1.4 |
| Body | 14px | 400 (regular) | 1.5 |
| Tool heading (h2) | 16px | 600 (semibold) | 1.25 |
| Section heading (h1) | 20px | 600 (semibold) | 1.25 |

Source: `globals.css` — body: `font-size: 14px; line-height: 1.5`. Label weights from `.field label`. Headings from `.workspace-heading h1 { font-size: 20px }` and `.tool-panel h2 { font-size: 16px }`.

Phase 5 specific:
- OCR table cell text: 12px / weight 400 — unified with label/caption tier
- Chart axis tick labels: 12px / weight 400 — Recharts `tick={{ fontSize: 12 }}` (consolidates former 11px)
- Chart title: 12px / weight 400 — color `var(--muted)` (consolidates former 13px)
- Chart type toggle buttons: 12px / weight 400 — matches `.quick-action-row` button sizing

## Color

All values from `globals.css` `:root`. No new tokens for Phase 5.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f7f8fa` (`--background`) | Page background, drop zone background, OCR result area bg |
| Secondary (30%) | `#ffffff` (`--surface`) | Tool panels, chat message list, file chip, result panels |
| Accent (10%) | `#0b6b57` (`--primary`) | Primary CTA buttons, active chart type button, copy button text, drop zone drag-over border, active nav item bg |
| Destructive | `#b42318` (`--destructive`) | Error messages only (form-error, OCR processing failure) |

Supporting tokens (from `globals.css`):
- `--sidebar: #1f2933` — sidebar (unchanged from Phase 1)
- `--text: #111827` — body copy, ghost button labels, OCR table cell text
- `--muted: #5e6a75` — placeholder text, subtitle copy, axis labels, metadata badges
- `--border: #d9dee5` — panel borders, drop zone border, table borders, chart toggle button border
- `--warning: #b54708` — quota warning banners (carried from Phase 2)
- `--info: #2f6fed` — info notices (carried from Phase 1)

Accent reserved for:
- "Enviar imagem" primary CTA button (OCR tool)
- "Sugerir Gráfico" quick-action button active state (border: `--primary`)
- Active chart type toggle button (background: `--primary`, text: white)
- Copy button text color (`color: var(--primary)`)
- Drop zone drag-over highlight border and tint background
- Focus-visible outline on all interactive elements (`outline: 2px solid var(--primary)`)

## Interaction States

### OCR Tool States

| State | Visual |
|-------|--------|
| `idle` | Upload panel visible, no file selected. Drop zone in default dashed state. CTA disabled. |
| `file_selected` | File chip appears below drop zone. CTA enabled ("Enviar imagem"). |
| `processing` | CTA shows "Processando..." and is disabled. Drop zone cursor: default. |
| `complete` | `OcrResultPanel` rendered with table + copy buttons. Upload panel hidden or collapsed. "Nova imagem" ghost button top-right of result panel. |
| `error` | `.form-error` banner below CTA. Error copy specific to failure type (see Copywriting). |
| `quota_exceeded` | `.note-block.warning` banner replaces CTA area. Pro upgrade link visible. |

### Chart States (inside ChatPanel)

| State | Visual |
|-------|--------|
| AI generating chart | Existing "Recebendo resposta..." streaming placeholder in `.chat-message-list` |
| Chart rendered | `ChartMessage` component appears as assistant message. Active type button highlighted in `--primary`. |
| Type switch | Instant local re-render — no loading state, no skeleton. |
| Copy chart data | "Copiar CSV" → "Copiado!" feedback, 1.5s revert. |

## Accessibility

Carried from existing project conventions (Phase 1–4):

- All interactive elements must have `focus-visible` outline: `2px solid var(--primary)` (from `globals.css`)
- Drop zone: `role="button"`, `tabIndex={0}`, `aria-label` describing action, keyboard activation via `Enter` or `Space`
- Copy buttons: `aria-label` includes format ("Copiar como TSV", "Copiar como CSV")
- Chart toggle buttons: `aria-pressed` on the active type button
- Chart container: `role="img"` + `aria-label` describing the chart (e.g. "Gráfico de barras: Valor por Nome")
- Loading states: `aria-live="polite"` on the message list (already present in `chat-panel.tsx`)
- OCR result table: `<th scope="col">` for headers; `<caption>` with "Tabela reconstruída da imagem"
