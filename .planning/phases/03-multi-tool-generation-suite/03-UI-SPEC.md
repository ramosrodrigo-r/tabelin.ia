# Phase 3 — UI Design Contract

## Design System

Source: Phase 1 UI-SPEC (approved) + `apps/web/src/styles/globals.css` (verified 2026-05-25)

| Property | Value |
|----------|-------|
| Tool | none — custom CSS with Tailwind utility tokens |
| Preset | not applicable |
| Component library | none (hand-rolled components using CSS custom properties) |
| Icon library | lucide-react (already in use: Braces, ScrollText, Regex icons wired in sidebar) |
| Font | Inter, ui-sans-serif, system-ui, -apple-system fallback chain |
| Layout posture | Operational SaaS workspace — dense, quiet, task-focused. No decorative chrome. |

- Font: `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", monospace`

Exactly 4 font sizes in use: 12px, 14px, 16px, 20px. Exactly 2 font weights in use: 400 (body/meta/code) and 600 (labels, headings, page titles).

No viewport-scaled font sizing. Letter spacing stays `0`.

**Safety warning banner (new for Phase 3):** Uses `--warning` (#B54708) for border-left accent and text, with `rgb(181 71 8 / 8%)` background — mirrors the existing `.quota-warning` pattern from Phase 2. Not a new color; reuses existing token.

**Pro upgrade CTA (Free users on Templates tool):** Uses `.quota-blocked` pattern from Phase 2 — neutral surface with `--border` and a primary-button CTA. No new color needed.

**Syntax highlighting palette:** Must use a low-saturation, muted theme compatible with the `#FBFCFD` output-box background. Recommended: GitHub Light theme or equivalent with colors drawn from the existing palette where possible. Avoid bright rainbow token colors that clash with the workspace's quiet posture.

## Tool-Specific Interaction Contracts

### Scripts Tool (`/workspace/scripts`)

**Layout:** Two-panel tool-grid (input left, output right). Mirrors Formula tool exactly.

**Input panel selectors:**
- Script type selector: segmented control (3 options, full width)
  - `VBA` — Excel automation
  - `Apps Script` — Google Sheets automation
  - `Airtable Script` — Airtable automation
- Text area: Portuguese-language prompt describing the automation task
- Submit button: "Gerar script" (pending: "Gerando...")

**Output panel:**
- Heading: "Script gerado"
- Status line (aria-live polite): "Recebendo resposta..." during streaming; "Pronto para revisar e copiar." when complete
- Metadata chip (below heading): script type label (VBA / Apps Script / Airtable Script)
- Code output: syntax-highlighted block using the script type language
  - VBA: VBA/Basic highlighting
  - Apps Script: JavaScript highlighting
  - Airtable Script: JavaScript highlighting
- Safety warning banner (`.note-block.warning`): displayed inline above code output if destructive operations detected
- Copy button: copies raw code text (no markdown fences)
- Assumptions block (`.note-block`): if AI returns assumptions

**States:** idle, loading, streaming, complete, error (same state machine as Formula)

### SQL Tool (`/workspace/sql`)

**Layout:** Two-panel tool-grid. Mirrors Formula tool.

**Input panel selectors:**
- Dialect selector: compact `<select>` element (5 options)
  - PostgreSQL
  - MySQL
  - SQL Server
  - Oracle
  - BigQuery
- Text area: Portuguese-language prompt describing the query
- Submit button: "Gerar SQL" (pending: "Gerando...")

**Output panel:**
- Heading: "Consulta gerada"
- Status line (aria-live polite): same pattern as Scripts
- Metadata chip: dialect name (ex: "PostgreSQL")
- Code output: SQL syntax highlighting
- Safety warning banner: displayed if `DROP`, `DELETE`, `TRUNCATE`, or `UPDATE` without `WHERE` is detected in output
- Copy button: copies raw SQL text (no markdown fences)
- Assumptions block: if AI returns assumptions

**States:** idle, loading, streaming, complete, error

### Regex Tool (`/workspace/regex`)

**Layout:** Two-panel tool-grid. Has two modes like Formula.

**Mode tabs:** "Gerar regex" | "Explicar regex"

**Generate mode input:**
- Text area: Portuguese-language prompt describing the pattern needed (ex: "CPF com pontos e traco")
- Submit button: "Gerar regex" (pending: "Gerando...")

**Explain mode input:**
- Text area: paste an existing regex pattern
- Submit button: "Explicar regex" (pending: "Explicando...")

**Output panel (generate):**
- Heading: "Regex gerada"
- Code output: regex highlighting (or plain monospace if highlighting library lacks regex support)
- Explanation text: plain Portuguese below the pattern
- Copy button: copies regex string only (not the explanation)

**Output panel (explain):**
- Heading: "Explicacao"
- List of step-by-step Portuguese explanations (ordered list, same as Formula explain mode)
- Copy button: copies full explanation text

**States:** idle, loading, streaming, complete, error

### Templates Tool (`/workspace/templates`) — Pro only

**Layout:** Two-panel tool-grid. Single mode (generation only).

**Free user state (upgrade gate):**
- Input panel shows text area and submit button visually
- Submit triggers the upgrade gate instead of generation
- Gate uses `.quota-blocked` pattern: description paragraph + "Assinar Pro" primary-button CTA
- No separate modal — inline in the input panel (same pattern as quota-blocked on Formula)

**Pro user input:**
- Text area: Portuguese-language description of desired spreadsheet (ex: "controle de gastos mensais")
- Submit button: "Gerar template" (pending: "Gerando...")

**Pro user output:**
- Heading: "Template gerado"
- Output rendered as formatted Markdown or CSV copy-ready block
- Copy button: copies full template text
- Assumptions block: if AI returns structural assumptions

**States:** idle, loading, streaming, complete, error (Pro) | idle, upgrade-gate (Free)

## Safety Warning Contract (SAFE-01)

**Trigger:** Inline banner appears above the code/query output block when a destructive operation is detected.

**Placement:** Inside `.output-panel`, between the metadata row and the `.output-box` code block. Never below the code.

**Visual:** Reuse `.note-block.warning` class (amber left-border, amber background tint, `--warning` text color). Add warning icon from lucide-react (`AlertTriangle`, 16px) before heading text.

**Heading text:** "Atenção — Operação destrutiva"

**Body text examples:**

| Trigger | Warning body |
|---------|-------------|
| SQL DROP / TRUNCATE | "Este script apaga dados permanentemente. Faça um backup antes de executar." |
| SQL DELETE sem WHERE | "DELETE sem WHERE apaga todas as linhas da tabela. Verifique a clausula WHERE antes de executar." |
| SQL UPDATE sem WHERE | "UPDATE sem WHERE altera todos os registros. Adicione uma clausula WHERE para limitar o impacto." |
| VBA DeleteFile / Kill | "Este script exclui arquivos do sistema. Confirme os caminhos antes de executar." |
| VBA/Apps Script deleteSheet / deleteRecord | "Esta operacao remove dados da planilha ou base permanentemente. Nao pode ser desfeita." |

**Copy button behavior:** Remains enabled. Warning does not block copy. User chooses to proceed.

**Icon:** `AlertTriangle` from lucide-react, 16px, `aria-hidden`, placed before heading.

## State Contract

Every new tool page (Scripts, SQL, Regex, Templates) must implement exactly these states:

| State | Description | Visual |
|-------|-------------|--------|
| `idle` | Tool loaded, no generation attempted | Output box shows idle message. Submit enabled. |
| `loading` | Request submitted, before first stream chunk | Spinner or status text "Preparando resposta...". Submit disabled. |
| `streaming` | Stream chunks arriving | Output box `data-status="streaming"` (blue border). Live text updates. Submit disabled. |
| `complete` | Output validated and ready | Output box `data-status="complete"` (green border). Copy button enabled. Safety banner if needed. |
| `error` | Generation or validation failed | Error block with message and retry button. Input preserved. |
| `quota-blocked` | Free user hit 4-use limit | Quota-blocked block replaces submit. Checkout CTA visible. |
| `pro-gate` | Templates tool, Free user | Upgrade gate in input panel. No generation possible. |

Streaming status must use `aria-live="polite"` on the status line element.

## Responsive Contract

Inherits Phase 1 rules. No changes for Phase 3.

- Desktop: `.tool-grid` is two-column (input 280–420px, output fills remaining width)
- Tablet/mobile (≤820px): `.tool-grid` stacks to single column
- Script type and SQL dialect selectors must not truncate labels on 375px viewport
- Safety warning banner must be readable at 375px (wraps naturally, no horizontal scroll)
- Syntax-highlighted code block uses `overflow-x: auto` to handle long lines without breaking layout

---

## Accessibility Contract

Inherits Phase 1 rules. Phase 3 additions:

- Script type selector (segmented control): `aria-pressed` on each segment button, `<fieldset>` + `<legend>` wrapping
- SQL dialect selector (`<select>`): associated `<label>`, standard focus ring
- Mode tabs on Regex tool: `role="tablist"` on container, `role="tab"` + `aria-selected` on each tab (same pattern as Formula)
- Safety warning banner: must not be the only indicator — include the `AlertTriangle` icon text alongside warning color
- Syntax highlighting: must not rely on color alone to convey meaning; code must be readable in forced-colors mode
- Pro gate on Templates: upgrade CTA button must be keyboard reachable and have descriptive accessible label "Assinar o plano Pro"
