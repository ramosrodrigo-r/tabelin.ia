<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fórmula de célula → formulajs | Conteúdo de célula que começa com "=" pode conter nomes de função arbitrários |
| motor de fórmulas → eval() | Motor NÃO deve usar eval() ou new Function() em nenhum ponto do fluxo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-FORMULA | Tampering | evaluateFormula — formula injection | mitigate | Motor só delega a funções explícitas do objeto formulajs via formulajs[enFnName]; nenhum eval() ou new Function() em qualquer ponto; função não mapeada → #NAME? (não executada). Mitigação: testes de evaluateFormula com "=DESCONHECIDA()" devem retornar "#NAME?" — verificado pelo teste "Função não mapeada retorna #NAME?" |
| T-14-CIRC | Spoofing | evaluateFormula — loop infinito | mitigate | Set evaluating detecta ciclos antes de stack overflow; célula circular retorna "#CIRC!" imediatamente. Mitigação: teste "fórmula circular retorna #CIRC!" no formula-engine.test.ts |
| T-14-PITFALL1 | Tampering | extractRange — VLOOKUP com flat array | mitigate | extractRange() sempre retorna (string|number)[][] 2D; teste unitário verifica shape antes de passar ao formulajs.VLOOKUP |
</threat_model>
