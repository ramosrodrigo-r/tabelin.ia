    Corpo: try { return await prisma.$transaction(async (tx) => { const count = await tx.conversationExchange.count({ where: { userId: input.userId, toolKind: input.toolKind } }); if (count >= 50) { const toDelete = await tx.conversationExchange.findMany({ where: { userId: input.userId, toolKind: input.toolKind }, orderBy: { createdAt: "asc" }, take: count - 49, select: { id: true } }); await tx.conversationExchange.deleteMany({ where: { id: { in: toDelete.map((r) => r.id) } } }); } return tx.conversationExchange.create({ data: { userId: input.userId, toolKind: input.toolKind, mode: input.mode, platform: input.platform ?? null, dialect: input.dialect ?? null, userPrompt: input.userPrompt, assistantPayload: input.assistantPayload as object } }); }); } catch { console.warn("ConversationExchange persistence skipped."); return null; }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| route handler → repository | Dados do usuário (prompt) e do LLM (payload) chegam ao repository após parse+validate no handler |
| repository → PostgreSQL | Queries Prisma com parâmetros tipados — nenhuma interpolação de string SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of Privilege (IDOR) | conversation-repository.ts — queries futuras de GET | mitigate | Toda query que filtra por id DEVE incluir userId no where clause (padrão IDOR guard do file-repository.ts). Phase 6 não cria endpoints GET, mas o repository deve seguir o padrão desde a criação para Phase 7. |
| T-06-02 | Tampering (SQL Injection) | assistantPayload — conteúdo LLM salvo como Json | accept | Prisma usa parameterized queries automaticamente; Json @db.Json não interpola valores como SQL raw. Risco residual mínimo. |
| T-06-03 | Information Disclosure (orphan records) | prisma/schema.prisma — relação User→ConversationExchange | mitigate | onDelete: Cascade declarado na relação — banco remove todos os exchanges ao deletar o User, sem lógica de aplicação (per PRIV-01). |
| T-06-04 | Denial of Service (race condition no cap) | conversation-repository.ts — count + delete + create | mitigate | prisma.$transaction(async (tx) => {...}) garante atomicidade — count e insert são na mesma transação, prevenindo acúmulo acima de 50 em requests simultâneos. |
