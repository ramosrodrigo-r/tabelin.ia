## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| usuário → célula editável → arquivo exportado | O conteúdo de célula é input controlado pelo usuário; ao virar CSV/XLSX e ser aberto no Excel/Sheets, cruza para um interpretador de fórmulas/macros |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering / Elevation of Privilege | `sanitizeCellForExport` em table-export.ts (CSV + XLSX) | mitigate | Prefixo `'` em toda célula iniciada por `= + - @ \t \r \n` (OWASP CSV Injection); regex `DANGEROUS_LEAD` aplicada por célula, testada na Task 1 |
| T-15-02 | Tampering | `buildXlsx` cell-objects | mitigate | Todas as células de dados gravadas como `{t:"s"}` (string explícita) via `aoa_to_sheet`; nunca deixar SheetJS inferir tipo (evita `=...` virar fórmula viva). Assert `ws["A1"].t === "s"` |
| T-15-03 | Tampering | Injeção via separador para iniciar célula com payload no meio | mitigate | Sanitização **por célula** (não por linha) + quoting RFC 4180 em `buildCsv` |
| T-15-04 | DoS / Tampering | ReDoS/prototype-pollution em `xlsx` parse | accept | Fora de escopo: esta fase usa só o caminho `write` (`aoa_to_sheet`/`writeFile`), que não parseia input externo. Nenhuma dep nova; 0.18.5 mantida |
| T-15-SC | Tampering | npm/pnpm installs | accept | Nenhum pacote novo instalado nesta fase (`xlsx@0.18.5` já presente; audit `Approved (no install)` no RESEARCH) |
