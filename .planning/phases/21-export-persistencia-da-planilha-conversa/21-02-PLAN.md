# Phase 21, Plan 2: Server Component Loading & UI Integration

Este plano cobre a conexão das queries do servidor diretamente nos Server Components (`WorkspaceLayout` e `WorkspacePage`), propagando o estado inicial para os componentes clientes e ajustando o comportamento de reset do chat para resetar a planilha viva de forma coerente.

## Tasks

### Task 1: Atualizar o WorkspaceLayout (Server Component)
Carregar o estado inicial da planilha diretamente no servidor.

- **File**: `apps/web/src/app/(workspace)/workspace/layout.tsx`
- **Changes**:
  - Importar `getActiveSpreadsheetSpec` do repositório de conversas.
  - No corpo da função `WorkspaceLayout`, obter o spec ativo da planilha do usuário (`getActiveSpreadsheetSpec(user.id)`).
  - Passar o spec retornado como prop `initialSpec` para o componente `<WorkspaceShell>`.

### Task 2: Atualizar o WorkspaceShell (Client Wrapper Component)
Propagar o spec ativo do layout para o provedor de estado da planilha.

- **File**: `apps/web/src/components/app/workspace-shell.tsx`
- **Changes**:
  - Atualizar `WorkspaceShell` para aceitar a prop opcional `initialSpec?: TableSpecPayload`.
  - Passar `initialSpec` como prop para `<WorkspaceStateProvider initialSpec={initialSpec}>`.

### Task 3: Atualizar o WorkspacePage (Server Component)
Carregar o histórico de conversas no servidor e injetar no chat unificado.

- **File**: `apps/web/src/app/(workspace)/workspace/page.tsx`
- **Changes**:
  - Importar `findUnifiedConversationExchanges` do repositório de conversas.
  - No corpo da função `WorkspacePage`, obter a lista de exchanges de conversa do usuário (`findUnifiedConversationExchanges(user.id)`).
  - Mapear a lista obtida de `ConversationExchange[]` do Prisma para o tipo `PersistedExchange[]` esperado pelo `UnifiedChatTool` (cuidado para serializar campos de data e payload adequadamente).
  - Passar a lista mapeada como prop `initialExchanges` para `<UnifiedChatTool initialExchanges={mappedExchanges} />`.

### Task 4: Sincronizar Limpeza da Conversa e Planilha no Cliente
Garantir que ao resetar/limpar a conversa, a planilha na tela também retorne ao estado inicial.

- **File**: `apps/web/src/features/unified-chat/unified-chat-tool.tsx`
- **Changes**:
  - No callback `handleNewConversation`, chamar `workspaceState.resetToSeed()` para redefinir a grade da planilha para a semente inicial (`SAMPLE_SPEC`).
