## Spacing Scale

Declared values (all multiples of 4, consistent with existing CSS):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gaps, inline badge padding, upload zone icon gap |
| sm | 8px | Column chip gap, chat message gap between role label and content |
| md | 16px | Panel internal padding (`tool-panel`), field-stack gap, chat input padding |
| lg | 24px | Workspace content padding (`workspace-content`), section breaks |
| xl | 32px | Auth panel padding (existing) — not used in this feature |
| 2xl | 48px | Major section breaks — not used in file-analysis layout |
| 3xl | 64px | Page-level — topbar height already 64px |

Exceptions:
- Upload drop zone minimum height: 120px (not a spacing token — it is a component minimum height).
- Chat message list: 8px gap between individual messages; 16px padding inside each bubble.
- Sheet selector button row: 8px gap between aba chips.
- Quick-action buttons ("Resumo Pivô" / "Relatório Executivo"): 8px gap between chips, 4px 12px chip padding.
- Touch targets for all interactive elements: minimum 40px height (consistent with `.primary-button` and `.nav-item`).

## Typography

Inherits existing body baseline from `globals.css`. No new font sizes or weights are introduced.

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px | 400 | 1.5 | Chat message content, schema preview descriptions, upload instructions |
| Label | 12px | 650 | 1.4 | Column type chips, section labels (`font-weight: 650` matches existing `.field label`), quick-action chip labels |

Accent (`--primary`) is reserved for:
1. "Enviar arquivo" primary button background
2. "Enviar mensagem" chat submit button background
3. Streaming/complete output-box border tint (inherits `.output-box[data-status="complete"]`)
4. Copy button text color on assistant messages
5. Active sidebar nav item background overlay (`rgb(11 107 87 / 28%)`)

Never use accent for: schema column chips, quick-action chips (use `--border` + `--surface`), sheet selector chips (use `--border` + `--surface`), or error states.

## Component Inventory

New components for this phase, following the `features/{tool}/components/` pattern from Phases 1–3.

### FileUploadPanel

**File:** `apps/web/src/features/file-analysis/components/file-upload-panel.tsx`

**Visual:** A `.tool-panel` container with a drag-and-drop drop zone.

Drop zone structure:
- Border: `1px dashed var(--border)`, `border-radius: 6px`, background: `#fbfcfd` (matches `.placeholder-box`)
- Minimum height: 120px
- Center-aligned content: `FileSpreadsheet` icon (lucide, 32px, `--muted` color) + heading "Arraste um arquivo ou clique para selecionar" (14px, `--muted`) + subtext ".csv ou .xlsx — máximo 5 MB" (12px, `--muted`)
- On hover/drag-over: border changes to `var(--primary)` tint (`rgb(11 107 87 / 40%)`)
- Hidden `<input type="file" accept=".csv,.xlsx">` triggered by click on zone

States:
- **idle:** Drop zone shown with icon + instructions + "Enviar arquivo" primary button (disabled until file selected)
- **selected (file chosen, not yet submitted):** File name chip appears below zone: filename + size in parentheses, `--border` outlined chip, `X` icon to clear. Primary button enabled.
- **uploading:** Button shows "Enviando..." with spinner (disabled). Drop zone shows "Enviando..." overlay.
- **error:** `.form-error` banner below zone with specific message (see Copywriting Contract).
- **needs-sheet-selection:** Drop zone hidden; `SheetSelector` appears in its place.
- **complete:** Drop zone hidden; `SchemaPreview` appears.

**One file at a time (D-09):** New upload clears current file state and resets chat. No multi-file support.

### SheetSelector

**File:** `apps/web/src/features/file-analysis/components/sheet-selector.tsx`

**Visual:** Inline in the tool panel, no modal. Appears after upload response for multi-sheet XLSX.

Structure:
- Heading: "Arquivo com múltiplas abas — escolha qual analisar" (14px, `--muted`)
- Row of chip buttons: one per sheet name. Chip style: `border: 1px solid var(--border)`, `border-radius: 6px`, `background: #fff`, `padding: 6px 12px`, `font-size: 14px`, `--text`
- Selected chip: `border-color: var(--primary)`, `color: var(--primary)`, `font-weight: 650`
- "Confirmar aba" primary button: appears after a chip is selected

### SchemaPreview

**File:** `apps/web/src/features/file-analysis/components/schema-preview.tsx`

**Visual:** Rendered as a system message at the top of the chat list — a visual `.note-block` variant with `border-left: 3px solid var(--info)` (matching existing `.note-block` style).

Content:
- Heading: "Arquivo carregado — {N} colunas detectadas" (14px, weight 650)
- Subtext: filename + total de linhas (12px, `--muted`)
- Chip row: one chip per column. Chip content: `{nome da coluna} · {tipo}`. Chip style: `border: 1px solid var(--border)`, `border-radius: 999px`, `background: #f8fafc`, `color: var(--muted)`, `font-size: 12px`, `font-weight: 650`, `padding: 4px 8px` — matches `.metadata-row span`
- Column types in pt-BR: `número`, `data`, `texto`, `booleano`

### ChatPanel

**File:** `apps/web/src/features/file-analysis/components/chat-panel.tsx`

**Visual:** Full-width section below the schema preview message. Replaces the two-column `.tool-grid` layout used by other tools. File Analysis uses a single-column chat layout within `.workspace-content`.

Layout within workspace-content:
```
[ SchemaPreview / welcome message — system bubble ]
[ ChatMessage (user) ]
[ ChatMessage (assistant) ]
[ ... ]
[ ChatInput row ]
[ QuickActionRow — "Resumo Pivô" + "Relatório Executivo" chips ]
```

Chat list: `display: grid`, `gap: 8px`, overflow-y scroll, min-height 300px, max-height calc(100vh - 280px).

### ChatMessage

**File:** `apps/web/src/features/file-analysis/components/chat-message.tsx`

**User message bubble:**
- `background: var(--primary)` tint `rgb(11 107 87 / 8%)`, `border: 1px solid rgb(11 107 87 / 20%)`, `border-radius: 6px`, `padding: 12px 16px`
- Aligned right — `margin-left: auto`, `max-width: 80%`
- Role label: "Você" (12px, `--muted`, above bubble)
- Content: 14px, `--text`

**Assistant message bubble:**
- `background: var(--surface)`, `border: 1px solid var(--border)`, `border-radius: 6px`, `padding: 12px 16px`
- Aligned left — full width or `max-width: 90%`
- Role label: "Tabelin.IA" (12px, `--muted`, above bubble)
- Content: rendered as `<pre style="white-space: pre-wrap">` for Markdown output (D-11 + RESEARCH.md Assumption A4: `<pre>` with `white-space: pre-wrap` is acceptable for MVP)
- CopyButton: appears in top-right corner of bubble once message status = complete. Uses existing `CopyButton` component from `features/formula/components/copy-button.tsx`

**Streaming assistant message:**
- `border-color: rgb(47 111 237 / 40%)` (matches `.output-box[data-status="streaming"]`)
- Status text below role label: "Recebendo resposta..." (12px, `--muted`, `aria-live="polite"`)
- CopyButton disabled during streaming

**System message (SchemaPreview rendered inline):**
- `border-left: 3px solid var(--info)`, `background: #f8fafc`, `border-radius: 0 6px 6px 0`, `padding: 12px 16px`
- No role label. No copy button.

### ChatInput

**Part of ChatPanel.** Input row at the bottom of the chat area.

Structure:
```
[ textarea (flex-grow) ] [ Enviar button ]
```
- `textarea`: `border: 1px solid var(--border)`, `border-radius: 6px`, `padding: 10px 12px`, `resize: none`, `rows={2}`, font inherited, placeholder "Faça uma pergunta sobre os dados..."
- On focus: `outline: 2px solid var(--primary)` (inherits focus-visible rule)
- "Enviar" button: `.primary-button`, min-height 40px, contains `Send` lucide icon (16px) + "Enviar" label, disabled when: no text, uploading, or streaming

### QuickActionRow

**Part of ChatPanel.** Chip row below the chat input.

Two chips, fixed after upload (D-10):

| Chip | Label | Prompt sent on click |
|------|-------|----------------------|
| Pivot | "Resumo Pivô" | Sends pre-built pivot prompt to chat route |
| Report | "Relatório Executivo" | Sends pre-built executive report prompt to chat route |

Chip style: `border: 1px solid var(--border)`, `border-radius: 6px`, `background: #fff`, `padding: 6px 16px`, `font-size: 13px`, `font-weight: 650`, `color: var(--text)` — secondary button style, visually distinct from but not competing with the primary "Enviar" button (D-10: "visually destacados but not intrusive").

On click: chip triggers `onSendQuickAction(prompt)` — same handler as chat input submit. Chip is disabled during streaming.

## Layout Override: File Analysis

All other workspace tools use `.tool-grid` (two-column: input panel left, output panel right).

File Analysis uses a **single-column chat layout** within `.workspace-content`:

```
.workspace-content
  .workspace-heading           ← "Análise de Arquivo" + description
  .file-analysis-layout        ← new CSS class, single column
    .tool-panel                ← FileUploadPanel (shown until upload complete)
      OR
    .chat-layout               ← shown after upload complete
      .schema-preview-bubble   ← system message at top
      .chat-message-list       ← scrollable message history
      .chat-input-row          ← textarea + send button
      .quick-action-row        ← Resumo Pivô + Relatório Executivo chips
```

CSS for `.file-analysis-layout`:
```css
.file-analysis-layout {
  display: grid;
  gap: 16px;
  max-width: 800px;
}

.chat-message-list {
  display: grid;
  gap: 8px;
  overflow-y: auto;
  min-height: 300px;
  max-height: calc(100vh - 280px);
  padding: 4px 0;
}

.chat-input-row {
  display: flex;
  gap: 8px;
  align-items: flex-end;
}

.quick-action-row {
  display: flex;
  gap: 8px;
  flex-wrap: wrap;
}
```

## Interaction States

### Chat States

| State | Visual |
|-------|--------|

| post-upload, no messages | Schema preview bubble + empty chat list + input enabled + quick action chips |
| user message sent | User bubble appended; assistant bubble appears with streaming border |

## Accessibility Requirements

- Upload zone `<div>` must have `role="button"` and `tabIndex={0}` with keyboard activation (Enter/Space triggers file picker).
- Upload zone label: `aria-label="Área de upload — arraste um arquivo ou pressione Enter para selecionar"`.
- Chat message list: `aria-live="polite"` on the list container so screen readers announce new messages.
- Chat streaming status: `aria-live="polite"` on status paragraph.
- Sheet selector chips: `aria-pressed` on each chip button.
- Quick action chips: `aria-label="Gerar resumo pivô"` and `aria-label="Gerar relatório executivo"`.
- Minimum touch target: 40px height for all interactive elements (matches existing `.primary-button` and `.nav-item`).
- Copy button: `aria-label="Copiar resultado"` / `"Copiado"` (already implemented in existing `CopyButton`).
