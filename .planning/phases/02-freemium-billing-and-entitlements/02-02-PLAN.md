<threat_model>
Assets: Mercado Pago access token, webhook secret, checkout external references, payment event IDs, entitlement state, Pro access.
Attackers: unauthenticated users creating checkouts for other users, clients forging return URLs or payment status, replayed/duplicate webhooks, forged webhook requests, accidental double activation from retries.
Required mitigations: authenticated checkout route, server-owned `external_reference`, provider credential isolation, optional webhook signature validation with raw body, unique provider event idempotency constraints, provider resource fetch before local entitlement mutation, return page that never grants Pro based only on query params.
Security verification: tests must prove unauthenticated checkout fails, invalid cycle fails, duplicate webhook event is idempotent, return URL status alone does not activate Pro, and provider access token is referenced only in server billing modules.
</threat_model>
