|----------|-----------|
| route handler → stream emitter | attachmentMeta construído com slice e length — conteúdo já validado pelo extractor (Phase 9/10) |
| stream emitter → client NDJSON | extractedText no evento é o próprio documento do usuário, isolado por userId+toolKind |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01-01 | Information Disclosure | extractedText no evento NDJSON | accept | É o documento do próprio usuário; isolamento por userId+toolKind garantido pelo route (session + findConversationExchanges por userId) — sem vazamento cross-user |
| T-11-01-02 | Tampering | attachmentMeta.extractedText construído no route | mitigate | Usar attachmentContext.slice(0, MAX_EXTRACTED_CHARS) — nunca passar o texto completo além do cap; route já valida size ≤5MB antes |
| T-11-01-03 | Denial of Service | extractedText grande no array de eventos NDJSON | mitigate | MAX_EXTRACTED_CHARS=8000 limita a 8KB por evento; bytes já validados no route (Phase 10) |
| T-11-01-SC | Tampering | sem novos pacotes npm nesta wave | accept | Nenhum pacote novo; Package Legitimacy Audit não aplicável (RESEARCH.md confirmado) |
