# Quick Task 260524-o3i: CSRF/origin auth hardening and one-use password reset

## Goal

Harden `POST /api/auth/*` against cross-origin form/fetch abuse and replace the placeholder forgot-password behavior with an expiring, single-use reset token stored as a hash in `Verification`.

## Tasks

1. Auth POST origin guard
   - Files: `apps/web/src/app/api/auth/[...all]/route.ts`
   - Action: validate `Origin` or `Referer` before mutating auth actions; allow only `BETTER_AUTH_URL` origin or the current request host origin.
   - Verify: sign-in, sign-up, sign-out, and forget-password reject a foreign origin.

2. Password reset token lifecycle
   - Files: `apps/web/src/server/auth/reset-password.ts`, `apps/web/src/app/api/auth/[...all]/route.ts`, `apps/web/src/app/(auth)/reset-password/page.tsx`
   - Action: generate random reset tokens, store only SHA-256 hashes in `Verification` with expiration, consume via `POST /api/auth/reset-password`, update the credential password, and delete/invalidate the token on use or expiry.
   - Verify: valid reset updates password and deletes token; expired and reused tokens fail.

3. Regression tests
   - Files: `apps/web/tests/auth-routes.test.ts`
   - Action: mock Prisma for auth-route tests and cover the CSRF and reset-token scenarios requested.
   - Verify: run the auth tests, then the web test suite or a scoped equivalent if unrelated failures block the full suite.
