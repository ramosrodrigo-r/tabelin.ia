# Quick Task 260525-s01: corrigir threats T-02-02-04 e T-02-03-03 da fase 2

## Objective

Close the two open Phase 2 security threats from `02-SECURITY.md` without changing the broader billing or Formula UX behavior.

## Tasks

### Task 1: Correct Mercado Pago webhook signature verification

**Files:** `apps/web/src/server/billing/webhook-service.ts`, `apps/web/src/app/api/billing/mercado-pago/webhook/route.ts`, `apps/web/tests/mercado-pago-webhook.test.ts`

**Action:** Replace raw-body-only HMAC validation with Mercado Pago's documented `x-signature` handling: parse `ts` and `v1`, build the manifest from `data.id` URL param when present, `x-request-id`, and `ts`, compare HMAC SHA-256 with constant-time comparison, enforce timestamp tolerance, and fail closed in production when `MERCADO_PAGO_WEBHOOK_SECRET` is absent. Route must pass the raw body, `x-signature`, `x-request-id`, and `request.url` into the service.

**Verify:** `pnpm --filter web test -- mercado-pago-webhook`

**Done:** Invalid signatures fail, stale signatures fail, valid signatures proceed, unsigned production webhooks fail closed, and existing invalid payload checks still work in test/dev.

### Task 2: Validate Pro support links before rendering

**Files:** `apps/web/src/server/support/support-config.ts`, `apps/web/src/components/app/topbar.tsx`, `.env.example`, `apps/web/tests/topbar.test.tsx`

**Action:** Add a small support config helper that normalizes the support email into a `mailto:` link only when the email is syntactically safe, normalizes WhatsApp URLs only when they are HTTPS and use an allowlisted WhatsApp host, and omits unsafe WhatsApp values. Update `Topbar` to render links from this helper instead of interpolating public env vars directly.

**Verify:** `pnpm --filter web test -- topbar`

**Done:** Pro users see safe email/WhatsApp links; Free users do not see support links; unsafe WhatsApp env values are not rendered.

### Task 3: Run focused regression checks

**Files:** `apps/web/tests/mercado-pago-webhook.test.ts`, `apps/web/tests/topbar.test.tsx`, `apps/web/tests/formula-ui.test.tsx`

**Action:** Run the focused webhook and UI checks, plus typecheck if changed types or imports require it.

**Verify:** `pnpm --filter web test -- mercado-pago-webhook && pnpm --filter web test -- topbar && pnpm --filter web test -- formula-ui && pnpm --filter web typecheck`

**Done:** All focused tests and typecheck pass.
